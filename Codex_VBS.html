CHAPTER 04 · 기획하기

04

여름성경학교 기획안 만들기

흩어진 자료를 읽고, 방향을 세우고, 제출 가능한 워드 문서로 완성한다.

총회 아동부 주제 자료와 교회 상황 자료를 한 폴더에 넣고 시작합니다.
코덱스는 자료를 요약하고, 기획 방향을 비교하고, 목차를 설계한 뒤 제출용 docx 기획안까지 만듭니다.
이 챕터의 핵심은 “빈 문서 작성”이 아니라 이미 가진 자료를 사역 기획으로 구조화하는 경험입니다.
📄 Test/chapter4 폴더에서 작업 — 여름성경학교 기획안 만들기 5단계
1단계// 자료 요약
이 폴더의 자료들을 모두 읽고, 올해 여름성경학교 기획에 필요한 핵심 정보를 요약해줘.
총회 주제, 주제말씀, 교회 기본정보, 지난해 회고, 올해 사역조건, 예산 중심으로 정리해줘.
2단계// 기획 방향 3안
요약한 내용을 바탕으로 올해 여름성경학교 기획 방향을 3가지 안으로 제안해줘.
각 안마다 주제 해석, 핵심 메시지, 프로그램 톤, 장점, 주의점을 표로 비교해줘.
3단계// 목차 설계
가장 현실적인 안 하나를 골라서 제출용 기획안 목차를 만들어줘.
목차에는 기획 의도, 목표, 일정표, 세부 프로그램, 인력, 예산, 안전관리, 보호자 안내, 후속 양육을 포함해줘.
4단계// 초안 작성
위 목차를 바탕으로 실제 제출용 기획안 초안을 작성해줘.
교역자와 부서장이 바로 검토할 수 있는 문체로 쓰고, 예산 초안의 주요 항목도 반영해줘.
5단계// DOCX 완성
06_기획안_양식.docx를 참고해서 작성한 내용을 제출용 기획안 docx로 완성해줘.

오늘 날짜 기준 TodoList로 준비 상황을 보고, 랜덤 1·2·3등 뽑기와 조 섞기로 레크리에이션을 돕고, 실시간 조별 점수판으로 상점과 벌점을 관리합니다.
핵심은 개발 지식이 아니라 필요한 운영 장면을 말로 설명하고, 코덱스가 작은 앱으로 바꾸는 경험입니다.

이렇게 모은 자료는 다음 성경학교 기획안의 가장 현실적인 출발점이 됩니다.
💌 Test/chapter7/01_롤링페이퍼 폴더에서 작업 — 롤링페이퍼 웹앱 만들기
Plan mode와 Computer Use를 함께 쓰는 실습
Plan mode는 코덱스가 먼저 작업 순서와 확인 지점을 세우게 하는 방식입니다. Google Sheets, Apps Script, 웹앱 배포처럼 단계가 많은 작업에서는 중간에 빠지는 것이 줄어듭니다. Computer Use는 코덱스가 브라우저 화면을 실제로 열고 메뉴 클릭, 코드 붙여넣기, 배포 화면 확인 같은 작업을 도와주는 기능입니다. 권한 승인처럼 계정 보안이 걸린 순간에는 사용자가 직접 클릭하고, 그 다음 단계부터 코덱스가 다시 이어갑니다.
1단계// Google Sheets 연동 롤링페이퍼
Plan mode로 먼저 전체 작업 계획을 세운 뒤 진행해줘. 필요하면 Computer Use와 인앱브라우저를 사용해서 Google Sheets와 Apps Script 배포 화면까지 실제로 열어 확인해줘.

2026_성경학교_롤링페이퍼_명단_샘플.xlsx를 읽고, 여러 핸드폰에서 동시에 입력할 수 있는 Google Sheets 연동 롤링페이퍼 웹앱을 만들어줘.
엑셀의 학년, 조, 이름, 역할 정보를 기본 명단으로 사용하고, 명단 인원수는 파일에 들어 있는 전체 데이터를 기준으로 계산해줘. 학생, 교사, 교역자 같은 역할별 인원도 자동 요약해줘.

Google Sheets에는 명단 시트와 메시지 시트를 만들어줘.
명단 시트는 학년, 조, 이름, 역할 컬럼으로 구성하고, 메시지 시트는 ID, 작성시간, 조, 작성자, 받는 사람, 받는 사람 역할, 태그, 메시지 컬럼으로 구성해줘.

Google Apps Script 웹앱으로 배포할 Code.gs와 브라우저 화면용 rolling_paper.html을 함께 만들어줘.
Code.gs에는 명단 불러오기, 메시지 불러오기, 메시지 저장, 초기 시트 생성, 동시 입력 충돌을 줄이기 위한 LockService 처리를 포함해줘.
rolling_paper.html에는 조별 사람 선택, 작성자 이름, 받는 사람, 메시지 내용, 감사/격려/기도제목 태그, 작성 시간 표시, 조별/개인별 모아보기, 새로고침, 인쇄 화면을 포함해줘.

여러 기기에서 입력한 메시지가 같은 Google Sheet에 저장되고, 새로고침하면 최신 메시지를 다시 불러오게 해줘.
Apps Script 안에서 실행될 때는 google.script.run 방식으로 Google Sheet와 통신하게 하고, 로컬 미리보기에서는 데모 모드로 동작하게 해줘.
화면 오른쪽에는 현재 웹앱 URL을 담은 QR 코드를 자동 생성해서 표시해줘. 핸드폰으로 QR을 찍으면 바로 접속할 수 있어야 해.

Computer Use로 Google Sheet를 만들고, 명단 시트와 메시지 시트를 구성하고, Apps Script 편집기에 Code.gs와 rolling_paper.html을 붙여 넣는 단계까지 도와줘.
웹앱 배포 설정은 실행 권한 나, 접근 권한 모든 사용자로 맞춰줘.
권한 승인이나 Google의 보안 경고처럼 사용자가 직접 승인해야 하는 단계가 나오면 멈추고, 어떤 버튼을 눌러야 하는지 짧게 안내해줘.

배포가 끝나면 웹앱 URL을 인앱브라우저에서 열어줘.
Google Sheets 연동 표시, 명단 로딩, QR 코드 표시, 테스트 메시지 저장, 메시지 시트 반영 여부를 확인해줘.
테스트 메시지는 확인 후 삭제해서 실습 자료를 깨끗하게 돌려줘.

마지막으로 README.md에 Google Sheet 주소, 핸드폰 접속용 웹앱 URL, 배포 방법, 권한 승인 주의점, QR 코드 사용법을 정리해줘.
여러 핸드폰에서 동시에 입력하려면?
처음부터 Google Sheets를 저장소로 두고 Apps Script 웹앱을 배포하는 방식이 가장 설명하기 쉽습니다. 참가자는 같은 링크나 QR 코드로 접속해 메시지를 남기고, 모든 입력은 하나의 Google Sheet에 모입니다. 코덱스는 Google Sheet 생성, Apps Script 코드 작성, 배포 안내, QR 코드, 저장 테스트까지 함께 도와줄 수 있습니다.
📝 Test/chapter7/02_피드백_설문 폴더에서 작업 — 피드백 설문문항 만들기
Browser Use로 Google Forms 만들기
Browser Use는 코덱스가 브라우저 안에서 실제 웹서비스를 열고, 화면을 읽고, 클릭과 입력을 도와주는 방식입니다. 설문문항처럼 먼저 내용을 검토해야 하는 작업은 1단계에서 문항을 확정하고, 2단계에서 Browser Use로 Google Forms 화면에 옮기는 흐름이 가장 안전합니다. 로그인이나 권한 확인처럼 계정 보안이 필요한 순간에는 사용자가 직접 승인하고, 코덱스가 그 다음 입력 작업을 이어갑니다.
1단계// 설문문항 초안 확인
01_설문_제작기준.md를 읽고, 성경학교 피드백 설문문항 초안을 먼저 만들어줘.
보호자용, 교사용, 어린이용으로 나누고, 각 설문은 5분 안에 응답할 수 있게 구성해줘.
객관식, 5점 척도, 단답형, 서술형을 섞고, 문항 번호, 문항, 응답 형식, 활용 목적을 표로 정리해줘.
Google Forms에 바로 옮기기 좋은 문항 목록도 따로 정리해줘.
아직 Google Forms를 만들지는 말고, 사용자가 문항을 확인하고 수정 요청할 수 있도록 여기서 멈춰줘.
2단계// Google Forms 제작
사용자가 확정한 성경학교 피드백 설문문항을 바탕으로 Google Forms 설문을 만들어줘.
Browser Use를 사용해서 Chrome 브라우저를 열고, Google Forms에 접속해 새 설문을 만들어줘.

설문 제목은 2026 성경학교 피드백 설문으로 하고, 설명에는 응답 목적과 예상 소요 시간 5분을 간단히 적어줘.
보호자용, 교사용, 어린이용 문항이 섞여 있다면 섹션을 나누어 입력해줘.
객관식, 체크박스, 5점 척도, 단답형, 장문형 등 문항의 의도에 맞는 Google Forms 질문 유형을 선택해줘.

필수 응답이 필요한 핵심 문항은 필수로 켜고, 서술형 소감 문항은 선택 응답으로 둬.
설문 제작 중 로그인이 필요하거나 권한 확인 화면이 나오면 멈추고, 사용자가 직접 처리해야 할 버튼을 짧게 안내해줘.

설문을 다 만든 뒤에는 미리보기 화면으로 열어 문항 순서, 질문 유형, 필수 여부가 맞는지 확인해줘.
완성된 Google Forms 편집 링크와 응답자 공유 링크를 정리해서 알려줘.

자동화 프롬프트// 매일 오전 7시 뉴스 클리핑
매일 오전 7시에 성경학교와 교회학교 사역자가 확인하면 좋을 최신 뉴스를 클리핑해줘.

관심 키워드는 여름성경학교, 교회학교, 아동부, 다음세대, 어린이 사역, 기독교 교육, 안전사고 예방, 교육 트렌드, AI 활용 교육으로 해줘.
광고성 글, 단순 행사 홍보, 중복 기사, 출처가 불분명한 글은 제외해줘.

결과는 Codex에서 바로 확인할 수 있게 정리해줘.
중요한 새 소식이 있으면 오늘의 핵심 뉴스 3~5개를 제목, 출처, 날짜, 한 줄 요약, 사역 적용 아이디어, 확인 링크로 정리해줘.

자동화 프롬프트// 매일 오전 7시 일일 브리핑
매일 오전 7시에 오늘 하루를 준비할 수 있는 일일 브리핑을 만들어줘.

브리핑에는 오늘 날짜와 요일, 오늘의 주요 일정, 준비해야 할 일, 마감이 가까운 일, 확인해야 할 연락, 오늘 우선순위 3가지를 포함해줘.
결과는 Codex에서 바로 확인하기 좋게 짧고 선명하게 정리해줘.
형식은 1. 오늘의 한 줄 요약, 2. 주요 일정, 3. 우선순위 3가지, 4. 확인할 것, 5. 미뤄도 되는 일 순서로 해줘.

특별히 확인할 내용이 없으면 '오늘은 특별히 긴급한 브리핑 항목이 없습니다'라고 알려줘.
외부 발송은 하지 말고, Codex 확인함 또는 현재 스레드에서 확인할 수 있게 해줘.
